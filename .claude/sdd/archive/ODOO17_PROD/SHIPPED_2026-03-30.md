## Summary

Ambiente de produção self-hosted para Odoo 17 entregue como projeto Docker Compose completo (`odoo-projeto/`). O setup orquestra três serviços isolados — PostgreSQL 15, Odoo 17 e Nginx 1.25 — com persistência de dados via volumes nomeados, configuração sensível via `.env`, e Nginx configurado com redirect HTTP→HTTPS, gzip, headers de segurança, cache de assets estáticos e bloqueio do Database Manager. O banco e a aplicação não expõem portas ao host; todo o tráfego externo passa pelo Nginx.

## Lessons Learned

### Process

- O fluxo BRAINSTORM → DEFINE → DESIGN → BUILD em ciclo único foi eficiente para um projeto de infraestrutura bem-definido. A clareza do DEFINE (score 14/15) eliminou ambiguidades durante o build.
- Definir YAGNI explicitamente no brainstorm (Redis, monitoring, multi-instância) evitou scope creep durante o design.

### Technical

- **Nginx não expande variáveis de shell** em arquivos `.conf` — `server_name ${ODOO_DOMAIN}` não funciona nativamente. Soluções: `envsubst`, Lua, ou edição manual. Para MVP, `server_name _;` (catch-all) foi usado.
- **`depends_on: condition: service_healthy`** no Compose v3+ resolve race condition entre Odoo e Postgres sem scripts de espera — preferível a loops `wait-for-it.sh`.
- **`proxy_mode = True`** no Odoo é obrigatório atrás de Nginx — sem ele, IPs dos clientes e geração de URLs ficam erradas.
- **`list_db = False`** em produção oculta a lista de bancos na tela de login — reduz superfície de ataque.

### Communication

- O brainstorm capturou cedo a decisão de SSL manual (vs. acme-companion), evitando retrabalho no design do Nginx.
- Confirmar "single server, não cluster" logo no brainstorm manteve o escopo enxuto.

### Tools & Libraries

- `postgres:15-alpine` e `nginx:1.25-alpine` — imagens Alpine reduzem superfície de ataque e tamanho de download.
- Docker Compose `healthcheck` nativo é suficiente para ordenação de serviços em single-server sem dependências externas.

## Recommendations for Future Work

| Área | Recomendação |
|------|--------------|
| SSL automático | Adicionar `nginx-proxy` + `acme-companion` como perfil opcional no Compose (`--profile ssl-auto`) |
| Backup | Adicionar script `backup.sh` com `pg_dump` + compressão e instrução de cron |
| Observabilidade | Adicionar stack de monitoramento opcional (Prometheus + Grafana) como segundo Compose file (`docker-compose.monitoring.yml`) |
| Segurança avançada | Configurar `fail2ban` no host para bloquear tentativas de bruteforce no Nginx |
| Longpolling workers | Configurar `gevent` workers separados para melhor performance de chat/notificações em produção com alto volume |
